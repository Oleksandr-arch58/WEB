# 🎮 MLBB Info Backend — Система аутентификации

## 📋 Описание

Полнофункциональная система аутентификации с хранением данных на диске:
- ✅ Регистрация и вход пользователей
- ✅ SHA-256 хеширование паролей
- ✅ Хранение БД в JSON файле (c:\UTM 2 курс\Web\lab4\db\users.json)
- ✅ REST API на Express.js
- ✅ Защита страниц от неавторизованных пользователей

## 🔐 Как работает аутентификация

### На клиенте (браузер):
1. `login.html` — форма входа/регистрации
2. `js/auth.js` — отправляет данные на backend API
3. `js/guard.js` — проверяет наличие токена, перенаправляет на login если его нет

### На сервере (Node.js):
1. `server.js` — слушает запросы на порту 3000
2. Получает данные → валидирует → хеширует пароль → сохраняет в БД
3. Возвращает токен клиенту

### В БД (db/users.json):
```json
[
  {
    "id": 1711900000000,
    "login": "testuser",
    "passwordHash": "a94a8fe5...(SHA-256 хеш)..",
    "createdAt": "2026-03-31T10:30:00.000Z"
  }
]
```

## 🛣️ API Endpoints

### POST `/api/auth/register`
Регистрация нового пользователя

**Request:**
```json
{
  "login": "myuser",
  "password": "mypassword123"
}
```

**Response:**
```json
{
  "success": true,
  "message": "Регистрация успешна!",
  "user": {
    "id": 1711900000000,
    "login": "myuser"
  }
}
```

### POST `/api/auth/login`
Вход в систему

**Request:**
```json
{
  "login": "myuser",
  "password": "mypassword123"
}
```

**Response:**
```json
{
  "success": true,
  "message": "Успешный вход!",
  "token": "a1b2c3d4e5...(32 байта токен)",
  "user": {
    "id": 1711900000000,
    "login": "myuser"
  }
}
```

### GET `/api/admin/users`
Получить всех пользователей (админ)

**Response:**
```json
{
  "success": true,
  "total": 5,
  "users": [
    {
      "id": 1711900000000,
      "login": "user1",
      "createdAt": "2026-03-31T10:30:00.000Z",
      "passwordHashFirst8": "a94a8fe5..."
    }
  ]
}
```

### DELETE `/api/admin/users/:id`
Удалить пользователя

**Response:**
```json
{
  "success": true,
  "message": "Пользователь удалён",
  "deleted": "username"
}
```

## 🐛 Решение проблем

### "Backend не запущен"
```
❌ Ошибка: Backend не запущен! Убедитесь, что backend запущен на http://localhost:3000
```

**Решение:**
```powershell
cd "c:\UTM 2 курс\Web\lab4"
npm install  # если зависимости не установлены
npm start    # запустить сервер
```

### "EADDRINUSE: address already in use :::3000"
**Решение:** Порт 3000 уже занят. Измените PORT в server.js:
```javascript
const PORT = 3001; // или другой свободный порт
```

### "Cannot find module 'express'"
**Решение:**
```powershell
npm install
```

### "db/users.json не создан"
**Решение:** Создайте папку и файл вручную:
```powershell
mkdir db
echo "[]" > db\users.json
```

## 🔒 Безопасность

- ✅ Пароли **никогда не передаются** по сети в открытом виде
- ✅ Пароли **хешируются** SHA-256 на сервере
- ✅ Хеши хранятся в БД, оригинальные пароли **забываются**
- ✅ CORS включен только для localhost (можно ограничить)
- ✅ Токены генерируются криптографически безопасным способом
